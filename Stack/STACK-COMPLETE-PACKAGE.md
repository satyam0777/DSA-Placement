## ğŸ’¡ COMMON MISTAKES TO AVOID

### **Mistake 1: Not Checking Empty Stack**
```cpp
// âŒ WRONG
int top = s.top();  // Crash if empty!

// âœ… CORRECT
if(!s.empty()) {
    int top = s.top();
}
```

### **Mistake 2: Forgetting to Reverse (NGR/NSR)**
```cpp
// âŒ WRONG - Going right to left but no reverse
for(int i = n-1; i >= 0; i--) { ... }
return v;  // Order is wrong!

// âœ… CORRECT
reverse(v.begin(), v.end());
return v;
```

### **Mistake 3: Wrong Pop Condition in Monotonic Stack**
```cpp
// For Next GREATER
while(!s.empty() && s.top() <= arr[i]) s.pop();  // âœ… Pop smaller

// For Next SMALLER
while(!s.empty() && s.top() >= arr[i]) s.pop();  // âœ… Pop greater
```

### **Mistake 4: Confusing Index vs Value**
```cpp
// Stock Span needs INDICES
stack<int> s;  // âœ… Store indices
span = i - s.top();  // Calculate distance

// vs

// NGR can use VALUES
stack<int> s;  // âœ… Store values
v.push_back(s.top());  // Return value
```

### **Mistake 5: Not Handling Edge Cases**
```cpp
// Always check:
if(s.empty()) return -1;     // Empty stack
if(n == 0) return result;     // Empty input
if(n == 1) return {-1};       // Single element
```
