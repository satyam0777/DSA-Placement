## üéØ PATTERN DECISION TREE

```
Heap Problem?
    ‚Üì
"K largest" or "K smallest"?
    ‚Üí YES: Heap of size K
    ‚Üí K LARGEST: MIN HEAP (remove smallest)
    ‚Üí K SMALLEST: MAX HEAP (remove largest)
    
    ‚Üì NO
    
"Median" or "running median"?
    ‚Üí YES: TWO HEAPS
    ‚Üí Max heap (smaller half)
    ‚Üí Min heap (larger half)
    
    ‚Üì NO
    
"Reorganize" or "rearrange with constraint"?
    ‚Üí YES: FREQUENCY + MAX HEAP
    ‚Üí Greedy approach
    ‚Üí Use most frequent first
    
    ‚Üì NO
    
"Schedule" with cooldown?
    ‚Üí YES: FREQUENCY + HEAP + CYCLES
    ‚Üí Process in cycles
    ‚Üí Track cooldown
    
    ‚Üì NO
    
"Merge K" or "K lists"?
    ‚Üí YES: MIN HEAP WITH TRACKING
    ‚Üí Track {value, listIndex, elementIndex}
    ‚Üí Process smallest
    
    ‚Üì NO
    
"Minimum cost to merge/connect"?
    ‚Üí YES: GREEDY + MIN HEAP
    ‚Üí Always connect/merge smallest
    ‚Üí Accumulate cost
```

## üí° COMMON MISTAKES TO AVOID

### **Mistake 1: Wrong Heap for K Problems**
```cpp
// ‚ùå WRONG - Using max heap for K largest
priority_queue<int> maxHeap; // Will keep largest, not Kth largest!

// ‚úÖ CORRECT - Using min heap for K largest
priority_queue<int, vector<int>, greater<int>> minHeap;
// Min heap size k ‚Üí top is Kth largest
```

### **Mistake 2: Not Checking Empty Heap**
```cpp
// ‚ùå WRONG
int top = pq.top(); // Crash if empty!
pq.pop();

// ‚úÖ CORRECT
if(!pq.empty()) {
    int top = pq.top();
    pq.pop();
}
```

### **Mistake 3: Median - Wrong Balance**
```cpp
// ‚úÖ CORRECT Balance for Median
// Always: maxHeap.size() == minHeap.size() OR maxHeap.size() == minHeap.size() + 1

if(minHeap.size() > maxHeap.size()) {
    maxHeap.push(minHeap.top());
    minHeap.pop();
}
```

### **Mistake 4: Custom Comparator Confusion**
```cpp
// For MIN heap of pairs
priority_queue<pair<int,int>, 
               vector<pair<int,int>>, 
               greater<pair<int,int>>> minHeap;
// ‚úÖ greater<> makes it min heap

// For MAX heap (default)
priority_queue<pair<int,int>> maxHeap;
// ‚úÖ Default is max heap
```

### **Mistake 5: Not Maintaining Heap Size**
```cpp
// K problems - maintain size k
for(int num : arr) {
    minHeap.push(num);
    
    if(minHeap.size() > k) {
        minHeap.pop(); // ‚úÖ Maintain size k
    }
}
```
